# App Store Screenshot Remediation (2.3.3)

Use this checklist to replace any screenshot set that Apple flags as not showing the app in use.

## Required principle

The majority of screenshots must show real, interactive UI states from the app's main workflow.

For Safari Escape, that means showing users:

- Receiving a universal link destination
- Choosing/opening a preferred browser
- Handling fallback and error states
- Configuring browser preference in Settings

## Recommended 6.7-inch shot plan (1290 x 2796)

Use 5 screenshots and keep all of them as real in-app screens:

1. Home screen with a pasted HTTPS destination and the Open Destination button enabled.
2. Home screen during active handoff (loading/opening state visible).
3. Home screen with handoff failure state and manual copy/open fallback visible.
4. Settings screen with Safari selected.
5. Settings screen with Chrome selected.

## Capture quality checklist

- Do not use marketing-only posters or text-only slides.
- Keep status bar and native UI visible.
- Avoid oversized decorative overlays that hide app functionality.
- Ensure key controls and result states are legible.
- Keep screenshot language aligned with your App Store localization.

## Before upload

- Verify all 6.7-inch screenshots are from the current build.
- Ensure at least 3 out of 5 clearly show the primary in-app flow (home + action/result states).
- Re-check iPad screenshot sets with the same standards.
- Upload in App Store Connect using "View All Sizes in Media Manager".
